## 🔧 Step-by-Step Build Process

### Option 1: Build AAB (Recommended for Play Store)

**AAB (Android App Bundle)** is the recommended format for Google Play Store. It allows Google to optimize APK delivery for different devices.

#### 1. Install EAS CLI (if not installed)

```bash
npm install -g eas-cli
```

#### 2. Login to Expo

```bash
eas login
```

**Credentials:**
- Email: Your Expo account email
- Password: Your Expo password

#### 3. Build Production AAB

```bash
eas build --platform android --profile production
```

**What happens:**
1. ✅ Code is uploaded to Expo servers
2. ✅ Dependencies are installed
3. ✅ Android App Bundle (.aab) is built
4. ✅ Signing is handled by Expo (or Play Store)
5. ✅ Build link is provided

**Build Time:** 10-20 minutes

#### 4. Download AAB

Once build completes:
1. Click the provided link
2. Download the `.aab` file
3. Save as: `pulsemate-v1.0.0-production.aab`

---

### Option 2: Build APK (For Testing Only)

**APK** is useful for testing but **NOT recommended** for Play Store submission. Use AAB instead.

#### Build Preview APK

```bash
eas build --platform android --profile preview
```

This creates an installable APK for testing on devices before submitting to Play Store.

## 📱 Upload to Play Store

### Step 1: Create Release

1. In Play Console, go to **Production** → **Releases**
2. Click **"Create new release"**
3. Choose signing option:
   - ✅ **Let Google manage your app signing key** (Recommended)
   - This generates signing key for you

### Step 2: Upload AAB

1. Click **"Upload"**
2. Select your `app-release.aab` file
3. Wait for upload to complete (1-2 minutes)
4. Google will scan the AAB automatically

## 🎯 Build Commands Quick Reference

### Production Build (AAB for Play Store)
```bash
# Full production build
eas build --platform android --profile production

# Clear cache and rebuild
eas build --platform android --profile production --clear-cache
```

### Preview Build (APK for Testing)
```bash
# Build installable APK
eas build --platform android --profile preview

# Build and auto-submit to internal testing
eas build --platform android --profile preview --auto-submit
```

### Check Build Status
```bash
# View all builds
eas build:list

# View specific build
eas build:view [BUILD_ID]
```

### Submit to Play Store (After building)
```bash
# Auto-submit AAB to Play Store
eas submit --platform android --latest

# Submit specific build
eas submit --platform android --id [BUILD_ID]
```

## 🔐 App Signing Options

### Option 1: Google Play App Signing (Recommended)
- ✅ Google manages your signing key
- ✅ Automatic signing for future releases
- ✅ Key stored securely by Google
- ✅ Easier to manage

**Setup:**
1. First upload to Play Console
2. Select "Let Google manage your app signing key"
3. Google generates and stores keys
4. Future builds use same key automatically

### Option 2: Expo-Managed Signing
- ✅ EAS generates and stores keys
- ✅ Keys stored in Expo's servers
- ✅ Consistent across all builds

**Setup:**
- Already configured in `eas.json`
- `"credentialsSource": "remote"`
- EAS handles everything automatically

### Option 3: Local Keystore (Advanced)
- Manual keystore management
- Not recommended for beginners
- Requires storing `.jks` file securely

## 🚨 Common Issues & Solutions

### Issue 1: Build Fails with "Out of Memory"
**Solution:**
```bash
# Increase memory in eas.json
"production": {
  "android": {
    "buildType": "app-bundle",
    "gradleCommand": ":app:bundleRelease",
    "credentialsSource": "remote",
    "config": "app/build.gradle"
  }
}
```

### Issue 2: "Missing google-services.json"
**Solution:**
- Ensure `google-services.json` exists in project root
- Download from Firebase Console if missing
- Add to `.gitignore` if it contains secrets

### Issue 3: Build succeeds but crashes on device
**Solution:**
1. Check API URL is production (not localhost)
2. Verify all environment variables set
3. Test with `eas build --profile preview` first
4. Check Crashlytics/Sentry for error logs

### Issue 4: Play Console rejects AAB
**Reasons:**
- Missing permissions in manifest
- Target SDK version too old
- Security vulnerabilities detected

**Solution:**
- Update dependencies: `npm update`
- Check Play Console error messages
- Fix issues and rebuild

## 🔄 Future Updates

### Increment Version for Updates

**Update app.json:**
```json
{
  "expo": {
    "version": "1.0.1",
    "android": {
      "versionCode": 3
    }
  }
}
```

**Version Rules:**
- `version`: Semantic versioning (1.0.1, 1.1.0, 2.0.0)
- `versionCode`: Integer, must increment (2 → 3 → 4)

**Build and Submit Update:**
```bash
eas build --platform android --profile production
eas submit --platform android --latest
```
